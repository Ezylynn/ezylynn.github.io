DNS Sinkhole

Summary

  • A DNS sinkhole is used to blocked malicious DNS request

What this is

  • Works by intercepting DNS queries, compare them to dynamic list of known malicious domains
  • If query attempts resolve to a bad domain, then route the traffic to a null route

How it works (high-level)

For general malicious/fishy websites

  • For example, there's a known bad website called malware.com points to 1.1.1.1
  • To stop traffic from coming there, we configure our DNS server to point to 127.0.0.1 (our local IP address) if receives a query from malware.com

For advertisements

  • Your device queries a DNS server to resolve domain names into IP address
  • However, websites also load third party ads and trackers, requiring additional DNS lookups to advertisers domains
  • A configured DNS sinkhole allow us to intercept and block the unwanted queries, returning a null IP address to our browser

Issues with DNS Sinkhole

  • Won't block all ads or malcious websites, ONLY stops DNS-based threats
  • Potential false positives or incompelete database of malicious/ads sites
  • No native alerting
